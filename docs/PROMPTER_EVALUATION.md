## 1. 강점

### 1.1 명확한 프로젝트 비전
- SQL 학습 도구라는 **목표가 명확**함
- 시각화, 안전한 실습 환경 등 **핵심 가치를 잘 정의**함
- 사용자(SQL 입문자) 관점을 고려한 기능 기획

### 1.2 기술적 수용력
- sql.js, WebAssembly 등 **제안된 기술을 빠르게 수용**
- AI가 제시한 여러 옵션 중 합리적 선택 (예: RETURNING + 폴백 방식)
- 코드를 이해하고 직접 수정하려는 시도 (reset 버튼 직접 구현)

### 1.3 적극적인 참여
- 문제 발생 시 **즉시 피드백** 제공 ("리셋 버튼 누르면 db가 사라져")
- 결과물에 대한 관심과 개선 의지

## 2. 개선이 필요한 영역

### 2.1 요구사항 명확성 부족 ⚠️

**문제점:**
```
❌ "디자인 깔끔하게 만들기"
❌ "챕터를 스마트하게 나누기"
❌ "기본 SQL 입력 칸 만들기"
```

**개선 방향:**
```
✅ "카드 레이아웃으로 변경, 카드 간격 16px, hover 시 그림자 효과"
✅ "DDL/DML 카테고리로 분리, 각 카드에 아이콘 추가"
✅ "상단에 readonly textarea, 높이 100px, 배경색 #f5f5f5"
```

**영향:** AI가 추측에 의존하게 되어 잘못된 구현 → 재작업 발생

---

### 2.2 잦은 요구사항 변경 ⚠️

**관찰된 패턴:**
| 순서 | 요청 | 결과 |
|------|------|------|
| 1 | "selectChapter.html 만들어줘" | 생성 |
| 2 | "파일명 바꿔줘" | 수정 |
| 3 | "별도 페이지 말고 index.html에서 링크로" | 삭제 후 재구현 |

```
WAI-ARIA 추가 → "ARIA 없애줘" → 제거
→ 최초 요청 시 명세했다면 1회로 완료 가능
```

**추정 낭비 시간:** 전체 개발의 약 20-30%

---

### 2.3 불완전한 테스트 후 보고 ⚠️

**사례:**
```
[보고] "리셋 버튼을 누르고 실행버튼을 누르면 db가 사라져"

[부족한 정보]
- 어떤 SQL을 실행했는가?
- 콘솔에 에러가 있었는가?
- 어떤 동작을 기대했는가?
```

**개선된 버그 리포트 예시:**
```
[환경] Chrome 120, Windows 11
[재현 순서]
1. CREATE 챕터 진입
2. 테이블 생성 SQL 실행
3. 초기화 버튼 클릭
4. 다시 실행 버튼 클릭

[기대 결과] 기본 SQL이 재실행되어 테이블 표시
[실제 결과] 다이어그램 영역 빈 상태, 콘솔에 "db is undefined" 에러
```

---

### 2.4 AI 결과물 검증 부족 ⚠️

**관찰:**
- AI가 수정한 코드를 **Undo로 되돌린 후** 동일 문제로 재문의
- 코드 변경사항을 **충분히 검토하지 않고** 적용

**발생한 상황:**
```
AI: script.js 수정 (SQLLib 변수 추가, resetDB 함수 개선)
사용자: [Undo로 되돌림]
사용자: "리셋 버튼 문제 해결해줘" (동일 이슈 재문의)
```

**권장:** 변경사항을 이해한 후 적용하거나, 이해되지 않으면 질문하기

---

### 2.5 프로젝트 관리 미흡 ⚠️

**관찰된 문제:**
- TODO.md가 있으나 **우선순위 명시 부족**
- 완료/미완료 기준이 **불명확**
- 기능 완성 전 새로운 기능 요청 (스코프 크립)

**현재 TODO 상태:**
```markdown
- [ ] Remaining: init/undo feature    ← 범위/완료 조건 불명확
- [ ] Add confirmation and safety      ← 구체적 스펙 없음
- [ ] Test reset flow                  ← 테스트 케이스 미정의
```

| 항목 | 실천 방법 |

## 6. 결론

### 긍정적 측면
- 프로젝트를 **끝까지 진행하려는 의지**가 있음
- AI 피드백을 **수용하고 개선**하려는 자세
- 기술적 결정에서 **합리적 판단력** 보유

### 개선 필요 측면
- 요구사항의 **사전 구체화** 필요
- **일관된 방향성** 유지 (잦은 변경 지양)
- AI 결과물에 대한 **검증 습관** 필요

### 최종 코멘트

> *"좋은 프롬프터는 AI에게 '무엇을 원하는지'가 아니라 '무엇이 필요한지'를 알려준다.  
> 현재 수준은 평균적이나, 의식적인 개선 노력으로 빠른 성장이 가능하다.
