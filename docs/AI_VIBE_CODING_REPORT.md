## 1. 개요

### 1.1 세미나 주제
**"AI 활용(바이브코딩) 생산성과 결과"**

본 보고서는 AI 코딩 어시스턴트(GitHub Copilot)를 활용한 실제 프로젝트 개발 경험을 기반으로, AI 협업 개발 방식의 생산성과 결과물 품질을 분석합니다.

### 1.2 바이브코딩(Vibe Coding)이란?
- **정의:** 개발자가 자연어로 의도를 설명하고, AI가 코드를 생성하며, 개발자가 검토·수정하는 협업 개발 방식
- **특징:** 
  - 대화형 인터페이스를 통한 점진적 개발
  - 즉각적인 피드백과 반복 개선
  - 컨텍스트 기반 코드 생성

## 3. AI 활용 개발 방식(바이브코딩)

### 3.1 개발 워크플로우

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  1. 요구사항  │───▶│  2. AI 대화  │───▶│  3. 코드 생성 │───▶│  4. 검토/수정 │
│    정의      │    │    & 설계   │    │    & 적용    │    │    & 테스트  │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       │                                                        │
       └────────────────── 반복 개선 ◀────────────────────────────┘
```

### 3.2 AI 활용 유형별 분류

#### 유형 1: 기능 구현 요청
```
[사용자] "INSERT에서 rowid를 한 번에 넣는 경우 처리해줘"

[AI 응답] INSERT ... RETURNING rowid 우선 시도 + 폴백 로직 구현
         - 코드 생성
         - 에러 핸들링
         - 테스트 가이드 제공
```

#### 유형 2: 버그 수정 요청
```
[사용자] "리셋 버튼을 누르고 실행버튼을 누르면 db가 사라져"

[AI 응답] 문제 원인 분석 (비동기 타이밍 이슈)
         - SQLLib 모듈 레벨 변수 도입
         - resetDB() 함수 async/await 처리
         - localStorage 삭제 후 재생성 로직
```

#### 유형 3: UI/UX 개선 요청
```
[사용자] "챕터를 스마트하게 나누고 카드 형태로 만들어줘"

[AI 응답] - HTML 카드 그리드 레이아웃
         - CSS 스타일링 (hover 효과, 반응형)
         - JavaScript 이벤트 핸들러
```

#### 유형 4: 설계 상담
```
[사용자] "DB 저장/복원이 가능한가?"

[AI 응답] - localStorage 기반 영속성 설명
         - export/import 옵션 제안
         - 구현 방법 및 제한사항 안내
```

### 3.3 대화 기반 개발의 특징

| 특징 | 설명 | 예시 |
|------|------|------|
| **점진적 구체화** | 추상적 요구사항 → 구체적 구현 | "디자인 깔끔하게" → 카드 레이아웃 |
| **실시간 피드백** | 구현 후 즉시 문제점 보고 → 수정 | 헤더 안보임 → z-index 수정 |
| **컨텍스트 유지** | 이전 대화 기반 연속 개발 | 챕터 UI → 플레이그라운드 연동 |
| **선택지 제공** | 여러 해결책 제시 → 사용자 선택 | 옵션 A/B/C 제안 |

## 4. 개발 과정 상세

### 4.1 개발 타임라인

```
Week 1 (11/19~)
├─ 프로젝트 초기 설정
├─ SQL 실행 엔진 구현 (sql.js 연동)
└─ 기본 다이어그램 렌더링

Week 2
├─ INSERT 다중행 처리 개선 (RETURNING rowid)
├─ DELETE/DROP 미리보기 및 확인 기능
└─ 영향받는 행 하이라이트

Week 3
├─ 챕터 선택 UI (카드 레이아웃)
├─ SQL 플레이그라운드 페이지 구성
└─ 챕터별 기본 SQL 로드 시스템

Week 4 (~12/13)
├─ DB 초기화 기능 구현
├─ 버그 수정 및 안정화
└─ 문서화 (SUMMARY.md, TODO.md)
```

### 4.2 AI 협업으로 구현된 주요 기능

#### 4.2.1 INSERT RETURNING 처리
**문제:** 다중행 INSERT 시 각 행의 rowid 추적 불가

**AI 제안 해결책:**
```javascript
// 1차 시도: RETURNING rowid 사용
const modifiedSql = sql.replace(/;\s*$/, '') + ' RETURNING rowid;';
const res = db.exec(modifiedSql);

// 2차 폴백: last_insert_rowid() + affected 계산
if (!handledByReturning) {
    db.run(sql);
    const lastRowId = db.exec("SELECT last_insert_rowid()")[0].values[0][0];
    // rowid 범위 계산...
}
```

#### 4.2.2 DELETE/DROP 미리보기
**요구사항:** 파괴적 명령 실행 전 확인 필요

**AI 구현 흐름:**
```
DELETE 입력 → 영향받을 rowid 조회 → 하이라이트 표시 
→ 확인 버튼 생성 → 사용자 확인 → 실제 실행
```

#### 4.2.3 드래그 가능한 다이어그램
```javascript
// AI가 생성한 드래그 로직
function startDrag(e) {
    isDragging = true;
    currentElement = e.target.closest('.table-diagram');
    offsetX = e.clientX - currentElement.offsetLeft;
    offsetY = e.clientY - currentElement.offsetTop;
}
```

### 4.3 문제 해결 사례

| 문제 | 원인 | AI 해결책 |
|------|------|-----------|
| 헤더 안보임 | z-index/position 문제 | CSS 구조 분석 후 수정 |
| 리셋 후 DB 사라짐 | 비동기 타이밍 | SQLLib 모듈화 + async/await |
| 카드 클릭 안됨 | 이벤트 핸들러 누락 | linkChapters.js 생성 |

- 트레이드오프 설명 (성능 vs 호환성)
- 권장 사항과 대안 제공

## 7. 한계점 및 개선 방향

### 7.1 AI 협업의 한계

| 한계 | 설명 | 권장 대응 |

## 8. 결론

### 8.1 핵심 인사이트

#### AI 협업 개발의 효과
1. **생산성 향상**: 보일러플레이트, 스타일링, 디버깅에서 50-80% 시간 절감
2. **학습 곡선 완화**: 새로운 기술(sql.js, WebAssembly) 빠른 적용
3. **품질 유지**: 에러 핸들링, 폴백 로직 등 견고한 코드 생성

#### 성공적인 바이브코딩을 위한 조건
1. **명확한 요구사항 전달**: 구체적인 문제 설명
2. **점진적 개발**: 작은 단위로 변경 → 검증 → 반복
3. **문서화 습관**: 맥락 유지를 위한 기록
4. **검토 문화**: 생성된 코드 이해 및 검증

### 8.2 수치 요약

| 항목 | 결과 |
|------|------|
| 개발 기간 | 4주 |
| 코드 라인 수 | ~800줄 |
| 구현 기능 | 8개 주요 기능 |
| 추정 시간 절감 | 60-70% |
| 코드 품질 | 양호 (리팩토링 여지 있음) |

### 8.3 최종 제언

> **"AI는 개발자를 대체하지 않지만, AI를 활용하는 개발자가 그렇지 않은 개발자를 대체할 수 있다."**

바이브코딩은 개발자의 생산성을 크게 향상시키는 도구입니다. 그러나 최종 책임과 품질 보증은 여전히 개발자의 몫입니다. AI를 "주니어 개발자 페어 프로그래머"로 활용하되, 아키텍처 설계와 핵심 비즈니스 로직은 개발자가 주도하는 것이 바람직합니다.
